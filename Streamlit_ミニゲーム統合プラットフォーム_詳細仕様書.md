# Streamlit ミニゲーム統合プラットフォーム 詳細仕様書

# 1. プロジェクト概要

## 目的
1つのStreamlitアプリから4種類の短時間ゲームを選択して遊べる統合プラットフォームを開発する。

**重要方針**
- ゲーム間のデータ共有は行わない
- 共通通貨・レベル・実績なし
- 各ゲームは単独で完結する
- プレイ時間は5～10分

---

# 2. システム構成

```
ホーム
├── AI入国審査官
├── ブラックマーケット
├── 博物館潜入
└── 癒し猫カフェ
```

- 設定（BGM・SE等は将来拡張）

# 3. AI入国審査官

## コンセプト
限られた情報からAIか人間かを見抜く推理ゲーム。

## ゲームフロー
1. 入国者生成
2. 書類確認
3. 質問（回数制限）
4. 制限時間内に判定
5. 正誤表示
6. 次の入国者

## UI
左：書類
中央：会話ログ
右：質問・判定ボタン
上：残り時間

## 書類
- パスポート
- 国籍
- 生年月日
- 指紋
- 写真
- ビザ

## 質問例
- 趣味
- 家族
- 好きな食べ物
- 昨日の出来事
- 子供時代

## 判定要素
- 書類の矛盾
- 会話の違和感
- 心理的揺さぶり
- 時間制限

## 勝利条件
一定人数以上を正しく判定。

# 4. ブラックマーケット

## コンセプト
価値が不明な商品をAIとの心理戦で競り落とし利益を最大化する。

## フロー
1. 商品提示
2. 情報購入
3. AI入札
4. プレイヤー入札
5. 落札
6. 利益計算

## 商品例
- 絵画
- 宝石
- USB
- 薬品
- 古書

## 情報
- 噂
- 簡易鑑定
- 市場価格

## AI性格
- コレクター
- 投資家
- 転売屋

## 勝利条件
最終利益が目標以上。

---

# 5. 博物館潜入

## コンセプト
複数ルートと特性を活用して宝石を盗み脱出する。

## フロー
1. 特性選択
2. マップ生成
3. 潜入
4. 宝石回収
5. 脱出

## マップ
- ランダム生成
- 複数ルート
- レーザー
- 監視カメラ
- 警備員

## 特性
|特性|効果|
|---|---|
|ハッカー|監視カメラ停止|
|エンジニア|レーザー停止|
|怪盗|足音無効|
|幻術師|警備員誘導|

※使用回数制限あり

## アイテム
- EMP
- 煙幕
- 鏡
- ドローン

## 勝利条件
宝石を持って脱出。

---

# 6. 癒し猫カフェ

## コンセプト
10営業日で高評価を目指す経営ゲーム。

## フロー
1. 方針決定
2. 営業
3. イベント
4. 結果
5. 次の日

## 営業設定
- 価格
- 広告
- 設備

## 猫
- 機嫌
- 疲労
- 人気

## 客
- 学生
- 家族
- 会社員
- 観光客

## イベント
- 雨
- テレビ
- SNS
- 猫脱走

## 勝利条件
10日終了時の総合評価。
